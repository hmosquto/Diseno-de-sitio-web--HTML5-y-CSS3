FUNDAMENTOS DE BASE DE DATOS

En este capítulo se presentan los conceptos fundamentales necesarios para dimensionar la importancia de las bases de datos relacional en las organizaciones, así como los conceptos y desarrollo de estudio de casos para analizarlas y diseñarlas lógicamente.

En este capítulo se presentan los conceptos fundamentales necesarios para dimensionar la importancia de las bases de datos relacional en las organizaciones, así como los conceptos y desarrollo de estudio de casos para analizarlas y diseñarlas lógicamente. Por otro lado, para abordar las problemáticas de la estructuración de los datos en una organización y el diseño adecuado de ésta, se presentarán y desarrollaran estudio de casos que son analizados y diseñados. Así mismo se abordarán desde el estudio y análisis de los formatos que utiliza la organización y que hacen de estos un material importante de recolección de información.

Por lo tanto, esta unidad presenta tres capítulos, donde el primero fundamenta en conceptos generales, los tipos, los gestores, su arquitectura para terminar en la planificación de proyectos de diseño de bases de datos. En el segundo capítulo, se fundamenta en los conceptos del modelo de datos lógico basado en la técnica entidad relación, el procedimiento para elaborarlo y el desarrollo de dos estudios de caso. Objetivo general

Comprender los diferentes modelos de bases de datos, y en concreto dominar el modelo relacional. Y ser capaz de realizar el modelo de una base de datos relacional, a partir de la especificación de requerimientos de un proyecto, comprendiendo y aplicando los conceptos y transformaciones implicados.

Obgetivo específico

Aplicar los conceptos de modelado lógico o conceptual de datos mediante el análisis inicial de un Caso propuesto en el cual se hace necesario identificar las entidades existentes y sus posibles relaciones y carnalidades.

Aprender a identificar las relaciones existentes que tiene una base de datos el cual el lector sabrá identificar cuáles son los tipos de relaciones que tiene una base de datos.

Adquirir conocimientos sobre la descripción y Construcción del Modelo lógico mediante la matriz de relaciones (Entidades, Atributos y atributos clave) y el diagrama ER resultante (realizado en StarUML).

CONTENIDOS

1) Que es una base de datos

En esta unidad se podrá apreciar que es una base de datos con definiciones y acompañado con fuentes bibliográficas para ampliar su conocimiento

2) Tipos de modelos

Se podrá apreciar todos los modelos de bases de datos existentes y así poder realizar una pequeña definición de cada uno de ello acompañada de fuentes bibliográficas para ampliar dichos conocimientos

3) Como relacionar una base de datos

Esta unidad será de suma importancia para usted ya que podrá aprender cómo llevar a cabo el proceso de relacionar las distintas bases de datos

4) Instalación de aplicativos

Esta sesión estudiara como podemos instalar los distintos programas para realizar la modelación de las bases de datos

5) Estudio de casos

Aplicaremos los conocimientos adquiridos mediante una serie de estudios de caso para mejorar así la compresión de los conocimientos
